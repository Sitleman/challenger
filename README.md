# challenger
Challenger — Telegram mini app для командных челленджей с помесячным циклом. Каждый месяц команда выбирает общий челлендж, участники ежедневно выкладывают фото/видео-подтверждения в чат, а выполнение засчитывается по реакциям коллег. Приложение показывает общий календарь прогресса, личный счёт и «зал славы» с победителями прошлых месяцев, по максимуму переиспользуя нативные возможности Telegram.

# 📄 ФИНАЛЬНЫЙ СКЕЛЕТ — 3 экрана

## Навигация (bottom nav, 3 вкладки)

```
🏆 Зал славы    |    🎯 Челлендж    |    💡 Идеи
   (слева)         (центр, main)       (справа)
```

По умолчанию открывается **центр** (Челлендж).

---

## 🎯 Экран «Челлендж» (центральный, главный)

Два состояния:

### Состояние 1 — не вступил
- Название челленджа
- Описание
- Даты / сколько дней идёт
- **Кнопка «Вступить»**
- (мелко) сколько уже участников

### Состояние 2 — вступил
- **Твой счёт:** крупно `22/23` + «пропусков: 1 из 3»
- **Статус сегодня** (явно, крупно):
  - ⬜ «Сегодня не закрыто» + подсказка «выложи фото/видео с #пруф в тему» + кнопка «Открыть тему»
  - 🟡 «Ждёт подтверждения (нужно ещё N реакций)»
  - 🟢 «Сегодня закрыто ✅»
- **Общий календарь-heatmap** (все участники, статусы 🟢/🟡/⬜ по дням)
- Кнопка «Перейти в тему»

💡 Мелочь: при старте нового месяца этот экран сам показывает карточку нового челленджа с кнопкой «Вступить» (всплывашка не нужна — состояние 1 и есть эта карточка).

---

## 🏆 Экран «Зал славы» (слева)

### Список (главный вид)
- Лента прошлых челленджей по месяцам:
  - аватар(ы) победител(я/ей)
  - название челленджа
  - месяц
- Сортировка: новые сверху

### Внутрь челленджа (по тапу)
- Название, описание, месяц
- Победитель(и) + все участники
- **Финальный календарь** того челленджа (кто как прошёл)
- Кнопка «Открыть тему» (ссылка на архивную/закрытую тему)
- (опц.) итоги: топ по дням, самый популярный пруф

---

## 💡 Экран «Идеи» (справа)

- Копилка идей на следующий челлендж (любой добавляет в любой момент)
- Список идей (текст + автор)
- Кнопка «Запустить голосование» → бот постит нативный **Telegram Poll**
- Админ отмечает победившую идею → она становится челленджем след. месяца

*(Профиль/бейджи как отдельного экрана нет — счёт и статус живут на «Челлендже», регалии — в «Зале славы». Так чище.)*

## 🤖 Бот (логика, вне экранов)

| Триггер | Действие |
|---|---|
| `#пруф` фото/видео в теме | 🟡 мягкий ОК, запись дня, старт окна подтверждения |
| Реакция на пруф | пересчёт; ≥2 → 🟢 |
| Окно закрылось без 2 реакций | ⬜ пропуск (−1 из 3) |
| 21:00 | пинг тем, у кого сегодня нет пруфа |
| Старт месяца | пост условия + список участников; **выдать участникам админку с title «участник MM.YYYY»**; открыть тему |
| Вступление в челлендж (в течение месяца) | выдать тег этому участнику |
| Конец месяца | итоги; **снять админку/теги у всех**; закрыть тему |
| «Запустить опрос» из аппа | создать Telegram Poll |

⚠️ **Спайк №0 (до всего):** проверить, что бот видит `message_reaction` на чужие сообщения в группе. Не работает → план Б: аппрув кнопкой в календаре/ленте внутри аппа.

## 🗂 Данные (минимум)

- **Challenge:** id, month, title, description, start/end, status, topic_id, winner(s)
- **Participant:** user_id, challenge_id, joined_at, tag_granted (bool)
- **Day:** user_id, challenge_id, date, message_id, status(🟡/🟢/⬜), reactions_count, confirm_deadline
- **Idea:** text, author, created_at, status(backlog / voting / chosen)

---

## 🎨 Оформление
Розово-фиолетовые градиенты, поддержка тёмной/светлой темы TG, акцент на аватары победителей (Зал славы) и крупный счёт `X/Y` (Челлендж).

---

## 📦 Стек проекта

```
Bot:       Go + telego                    (long polling на старте)
Backend:   Go + chi                        (бот и API в одном бинаре)
DB:        PostgreSQL + sqlc               (пишем SQL → типобезопасный Go)
Jobs:      robfig/cron + горутины          (пинг 21:00, окна подтверждения)
Frontend:  React + Vite + TS
UI-kit:    @telegram-apps/telegram-ui      (нативные TG-компоненты)
Auth:      валидация initData на бэке
Deploy:    один Go-бинарь + Postgres (Railway/VPS)
```

**Ключевые факты, которые уже выяснили:**
- Реакции доступны боту **только как события** `message_reaction` (не как состояние) → счётчик ведём сами в БД
- Обязательно указать `allowed_updates: ["message", "message_reaction"]` — иначе события не приходят
- Бот должен быть **админом** группы
- Long polling досылает пропущенные апдейты при переподключении (~24ч буфер)
